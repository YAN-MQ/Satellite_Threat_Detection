# 工作汇报

## 1. 工作目标

本阶段工作的目标，是在已有 `ns-3 + 流量特征提取 + 深度学习分类` 工程基础上，完成一条可复现的网络威胁检测研究链路，并进一步扩展到面向低轨卫星多星协同场景的联邦学习版本。

整体工作分为三部分：

1. 数据与仿真链路整理
2. 单体模型训练与扩展实验
3. 联邦学习版本 `OrbitShield_FL` 的设计与实现

## 2. 数据仿真与预处理工作

### 2.1 数据来源与论文对齐

首先通读了数据集构建论文 《Toward Generating a New Intrusion Detection Dataset and IntrusionTraffic Characterization》，对照论文中的 CICIDS2017 数据集攻击类型和时间窗口，核查了原工程中数据标签和时间段的对应关系。

整理出当前稳定可用的三分类任务：

- `benign = 0`
- `ddos = 1`
- `portscan = 2`



### 2.2 数据处理链路

当前已经跑通的完整数据链路为：

`原始 PCAP -> 攻击时间窗口提取 -> IP 分片 -> NS-3 实时仿真 -> 流量捕获 -> 特征提取 -> dataset_window`

具体工作包括：

- 重写并修正时间窗口提取与分片脚本
- 修正 NS-3 实时仿真运行方式
- 重新组织 TAP 抓包流程
- 重新抽取仿真后流量并提取特征

最终生成了以下关键数据目录：

- [fragments_window](./fragments_window)
- [captured_window](./captured_window)
- [dataset_window](./dataset_window)

### 2.3 特征提取与数据集构造

现有数据集保留了固定的 18 维特征，不对原始定义做改动：

最终送入模型的 18 维特征如下：

| 序号 | 特征名 | 含义 |
|------|--------|------|
| `1` | `IAT` | 当前包与前一包的到达间隔 |
| `2` | `size` | 当前包长度 |
| `3` | `proto` | IP 协议号 |
| `4` | `SYN` | TCP SYN 标志 |
| `5` | `ACK` | TCP ACK 标志 |
| `6` | `RST` | TCP RST 标志 |
| `7` | `FIN` | TCP FIN 标志 |
| `8` | `PSH` | TCP PSH 标志 |
| `9` | `Size_M` | 近窗口包长均值 |
| `10` | `Size_S` | 近窗口包长标准差 |
| `11` | `IAT_M` | 近窗口到达间隔均值 |
| `12` | `IAT_S` | 近窗口到达间隔标准差 |
| `13` | `IAT_X` | 近窗口到达间隔最大值 |
| `14` | `IAT_N` | 近窗口到达间隔最小值 |
| `15` | `PPS` | 窗口内每秒包数 |
| `16` | `BPS` | 窗口内每秒字节数 |
| `17` | `SYN_R` | 窗口内 SYN 包比例 |
| `18` | `ACK_R` | 窗口内 ACK 包比例 |

数据预处理方式为：

- 窗口大小 `window size = 10`
- 滑动步长 `stride = 1`
- 按类别、按时间顺序切分 `train / val / test = 60 / 20 / 20`
- 先切分再归一化
- 最终输入形状为 `(samples, 10, 18)`

这部分工作确保了后续单体模型和联邦模型可以直接复用同一份 `dataset_window`。

## 3. 单体模型训练工作

### 3.1 主模型与训练流程

单体模型部分保留并复用了现有主模型 `DSC-CBAM-GRU`，其结构为：

`Input(10x18) -> Conv1D -> DSC -> CBAM -> GRU -> FC -> 3类输出`

训练过程采用：

- `CrossEntropyLoss`
- `AdamW`
- 三分类输出


在重新整理数据链路后，我基于新的 `dataset_window` 完整重跑了训练流程，并得到当前主模型结果：

- 最佳验证精度：`Val Acc = 0.9902`

## 4. 联邦学习扩展工作

### 4.1 工作动机

在单体模型达到较高精度后，我进一步考虑了更贴近低轨卫星场景的协同训练问题。

低轨卫星网络中的典型特征是：

- 单星本地样本有限
- 不同卫星观测流量分布存在非 IID 特性
- 轨道面内链路较稳定
- 轨道面间链路更动态、更易断续

因此，我在不破坏现有单机训练流程的前提下，新增实现了联邦学习版本，用于模拟多星协同威胁预测。

### 4.2 联邦版本总体设计

联邦版本最终命名为：

- `OrbitShield_FL`

其设计原则是：

- 不重写现有单机工程
- 直接复用当前 `dataset_window`
- 直接复用当前 `DSC-CBAM-GRU`
- 采用“算法仿真级联邦”，先在单机上模拟多卫星训练

当前默认星座设置为：

- 12 颗卫星
- 3 个轨道面
- 每个轨道面 4 颗卫星

即：

- `plane_0`: `sat_0 ~ sat_3`
- `plane_1`: `sat_4 ~ sat_7`
- `plane_2`: `sat_8 ~ sat_11`

### 4.3 联邦方法

本阶段联邦学习部分采用的是：

- `DFedSat` 风格骨架
- 面内聚合
- 面间异步 gossip
- 链路失败补偿
- 陈旧度衰减
- 信誉感知聚合

具体做法是：

1. 使用 `Dirichlet(alpha=0.3)` 将训练集划分到各颗卫星
2. 每颗卫星本地训练一个 `DSC-CBAM-GRU`
3. 同一轨道面内先做加权聚合
4. 轨道面间根据可见窗口做异步 gossip
5. 当邻面失联时，用历史模型与本面模型做补偿
6. 根据更新方向、验证损失改善和参与稳定性更新节点信誉
7. 用全局验证集和测试集评估每轮联邦模型

在工程实现上，我新增了联邦模块目录：

- [4_train/federated](./4_train/federated)

并实现了以下核心模块：

- `partition.py`：联邦数据划分
- `client.py`：卫星客户端抽象
- `topology.py`：动态拓扑模拟
- `aggregators.py`：面内聚合与状态字典运算
- `gossip.py`：面间异步 gossip
- `compensation.py`：链路失败补偿
- `reputation.py`：信誉更新
- `metrics_fl.py`：联邦评估指标
- `serverless_orchestrator.py`：联邦主控编排

### 4.4 联邦实验结果

我先跑通了多组联邦方法对比，包括：

- `single`
- `fedavg`
- `intra_only`
- `intra_gossip`
- `OrbitShield_FL`

在此基础上，又对完整方案进行了系统网格搜索，最终把正式版本命名为 `OrbitShield_FL`。

当前最终结果为：

- `Accuracy = 0.9718`
- `Precision = 0.9719`
- `Recall = 0.9718`
- `F1 = 0.9718`

相较历史联邦方法结果：

| 方法 | Accuracy | F1 |
|------|----------|----|
| `single` | `0.9601` | `0.9599` |
| `fedavg` | `0.9571` | `0.9570` |
| `intra_only` | `0.9636` | `0.9636` |
| `intra_gossip` | `0.9159` | `0.9143` |
| `OrbitShield_FL` | `0.9718` | `0.9718` |

可以看到，最终优化后的 `OrbitShield_FL` 已经优于：

- 单机集中式兼容基线 `single`
- 标准 `FedAvg`
- 仅面内聚合 `intra_only`

这说明在合理控制跨面信息交换强度的前提下，多星协同训练可以有效带来性能增益。

### 4.5 联邦调优结论

在网格搜索中，我重点调节了以下参数：

- `beta`
- `warmup_rounds`
- `global_momentum`

最终得到当前最优配置：

- `beta = 0.1`
- `warmup_rounds = 2`
- `global_momentum = 0.1`
- `beta_floor = 0.05`

该配置已被固化为联邦默认 demo 配置。
